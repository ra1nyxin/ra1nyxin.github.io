# Backup and Ransomware Resilience

备份要关注完整性、隔离性、可恢复性和验证结果。面对勒索、误删、供应链事故和管理员账号失陷，单独有一份数据副本并不够，恢复时间也必须满足业务要求。

## 1. 关键概念

| 概念 | 含义 |
| --- | --- |
| RPO | 最多可丢失多少数据 |
| RTO | 多久必须恢复服务 |
| Immutable backup | 攻击者难以修改或删除的备份 |
| Offline copy | 与生产凭证和网络隔离的副本 |
| Restore test | 实际恢复验证 |

没有恢复演练的备份，只能算希望。

## 2. 备份策略

建议采用分层策略：

- 生产快照用于快速回滚。
- 跨账号或跨区域备份用于灾难恢复。
- 不可变备份用于抵抗勒索。
- 离线副本用于极端场景。
- 关键配置和密钥也纳入备份计划。

数据库、对象存储、配置、身份系统和 CI/CD 配置都要考虑备份。

## 3. 权限隔离

备份系统必须和生产权限隔离。

要求：

- 生产管理员不能删除不可变备份。
- 备份删除策略需要独立审批。
- 备份账号启用强 MFA。
- 备份访问日志进入安全平台。
- 恢复权限和删除权限分离。

如果攻击者拿到生产管理员后能删除所有备份，备份无法抵抗勒索。

## 4. 恢复优先级

恢复顺序应提前定义：

1. 身份和访问控制。
2. 网络和基础设施控制面。
3. 核心数据库。
4. 关键业务服务。
5. 日志和监控。
6. 周边系统。

恢复身份系统尤其关键，否则其他系统可能无法安全启动。

## 5. 恢复演练

演练要验证：

- 备份是否可用。
- 恢复步骤是否完整。
- 所需权限是否可获得。
- 恢复时间是否满足 RTO。
- 恢复后数据是否一致。
- 恢复系统是否仍然安全。

演练结果要记录问题和修复动作。

## 6. 勒索防护

关键控制：

- 终端和服务器最小权限。
- 横向移动检测。
- 不可变备份。
- 备份账号隔离。
- 文件异常加密行为检测。
- 大规模删除告警。
- EDR 隔离能力。

勒索响应中不要急于恢复到可能已被植入后门的环境。

## 7. 数据完整性

备份完整性要求：

- 定期校验 hash 或快照状态。
- 记录备份任务成功率。
- 对关键数据库做一致性检查。
- 恢复后执行应用级验证。
- 监控备份容量异常变化。

备份成功日志不等于业务数据可恢复。

## 8. 事件响应衔接

勒索或破坏事件中，恢复前要确认：

- 初始访问路径是否关闭。
- 高权限凭证是否轮换。
- 持久化是否清理。
- 恢复点是否早于入侵时间。
- 恢复后是否增强监控。

否则恢复只是把攻击者重新带回生产。

## 9. 指标

| 指标 | 目标 |
| --- | --- |
| 关键系统备份覆盖率 | 100% |
| 不可变备份覆盖率 | 高价值系统 100% |
| 恢复演练通过率 | 持续提升 |
| RTO 达标率 | 可证明 |
| 备份删除告警 | 全覆盖 |
| 备份失败未处理数量 | 0 |

备份韧性的目标是让组织在最坏情况下仍然有可信恢复路径。
