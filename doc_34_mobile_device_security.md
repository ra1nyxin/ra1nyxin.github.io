# Mobile Device Security

移动设备同时承载身份、邮件、聊天、文件和 MFA，是现代办公中的高价值入口。移动安全需要在用户体验、隐私和企业数据保护之间取得平衡。

## 1. 管理模型

| 模型 | 适用 |
| --- | --- |
| MDM | 企业设备，控制更强 |
| MAM | BYOD，保护应用和数据 |
| Container | 企业数据隔离 |
| Conditional access | 根据设备状态决定访问 |

BYOD 场景下要避免过度收集个人隐私数据。

## 2. 基线要求

移动设备应满足：

- 屏幕锁和生物识别。
- 设备加密。
- 系统版本最低要求。
- 禁止越狱或 root。
- 企业应用受控分发。
- 丢失后可远程擦除企业数据。
- 高风险设备阻断访问。

设备不合规时，应限制访问企业数据。

## 3. 应用保护

企业应用策略：

- 禁止复制到个人应用。
- 限制截图或水印。
- 控制文件打开方式。
- 应用数据加密。
- 会话超时。
- 高风险操作重新认证。

应用保护比整机控制更适合 BYOD。

## 4. 移动身份风险

常见风险：

- MFA push 疲劳。
- SIM swap。
- 恶意配置描述文件。
- 钓鱼页面窃取会话。
- 恶意应用读取剪贴板。
- 设备丢失导致数据泄露。

高权限账号应优先使用抗钓鱼认证，而不是只依赖手机推送。

## 5. 日志和检测

关注：

- 新设备注册。
- 设备合规状态变化。
- 越狱或 root 检测。
- 企业数据擦除。
- 移动端异常登录。
- MFA 高频拒绝或通过。

移动安全日志要和身份日志联动。

## 6. 丢失设备响应

流程：

1. 确认设备和用户。
2. 标记设备不可信。
3. 撤销会话。
4. 远程擦除企业数据。
5. 检查近期登录和数据访问。
6. 重新注册新设备。

只擦除设备不够，还要处理会话和 token。

## 7. 隐私边界

BYOD 应明确：

- 企业能看到什么。
- 企业不能看到什么。
- 擦除范围是企业数据还是整机。
- 数据保留多久。
- 用户如何退出管理。

透明边界能降低推行阻力。

## 8. 指标

| 指标 | 目标 |
| --- | --- |
| 合规设备比例 | 持续提升 |
| 越狱/root 设备 | 0 |
| 高权限账号移动强认证 | 100% |
| 丢失设备处置时间 | 持续降低 |
| BYOD 隐私告知覆盖 | 100% |

移动安全的重点是保护企业数据和身份，不是无限控制用户设备。
