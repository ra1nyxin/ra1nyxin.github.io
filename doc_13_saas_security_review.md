# SaaS Security Review

企业越来越多核心数据存在 SaaS 里。SaaS 安全不只是采购评估，还包括接入、授权、配置、日志、数据流、第三方应用和退出策略。

一个 SaaS 一旦接入 SSO、同步用户、读取文件或承载客户数据，就应该进入安全治理范围。

## 1. 评估维度

| 维度 | 问题 |
| --- | --- |
| Identity | 是否支持 SSO、SCIM、MFA、角色管理 |
| Data | 存储什么数据、是否支持加密和导出控制 |
| Logging | 是否提供足够审计日志 |
| Admin | 管理员权限是否可分离 |
| Integration | 是否支持 API token、Webhook、OAuth 应用 |
| Compliance | 是否满足合同、地区和行业要求 |
| Resilience | 是否有备份、恢复和退出方案 |

不要只看供应商安全白皮书。要看实际租户中能配置什么、能审计什么。

## 2. 接入前检查

| 项目 | 要求 |
| --- | --- |
| SSO | 强制企业身份登录 |
| SCIM | 支持自动开通和离职回收 |
| Role model | 支持最小权限角色 |
| Audit log | 管理、登录、数据访问可审计 |
| Data residency | 符合数据地区要求 |
| API security | token 可限制范围并可轮换 |
| DPA / contract | 数据处理责任明确 |

如果一个 SaaS 不能提供审计日志，需要降低其承载数据等级。

## 3. 管理员权限

SaaS 管理员常被低估。

管理员治理要求：

- 管理员账号数量最小化。
- 管理员使用强 MFA。
- 权限分离，例如 billing、security、user admin。
- 管理操作进入日志。
- 管理员变更触发告警。
- 使用 break-glass 管理员时立即通知。

全局管理员不应作为日常工作账号使用。

## 4. OAuth 与第三方应用

SaaS 生态中最大的隐性风险之一是 OAuth 授权。

需要关注：

- 谁授权了应用。
- 应用能访问哪些数据。
- 是否能离线访问。
- token 是否长期有效。
- 应用发布方是否可信。
- 是否支持集中撤销。

高风险 scope 需要审批，例如读取全部邮件、访问所有文件、管理用户、导出审计日志。

## 5. 数据控制

SaaS 数据安全关注：

- 是否能限制分享范围。
- 是否能禁止公开链接。
- 是否能控制外部协作者。
- 是否能限制下载和导出。
- 是否支持 DLP 或内容扫描。
- 删除后是否同步清理备份。

对承载敏感数据的 SaaS，需要定期检查公开链接、外部共享和未使用集成。

## 6. 日志接入

优先接入这些日志：

| 日志 | 价值 |
| --- | --- |
| Login events | 异常登录和账号接管 |
| Admin activity | 配置变更和权限扩张 |
| Data access | 敏感数据读取 |
| Sharing events | 外部共享和公开链接 |
| API token events | 自动化访问风险 |
| Integration changes | 第三方应用风险 |

SaaS 日志常有延迟和 API 限速，检测规则要考虑这个现实。

## 7. 配置基线

常见基线：

- 强制 SSO。
- 禁止本地密码登录，除非作为受控应急。
- 禁止普通用户安装高权限应用。
- 默认关闭公开分享。
- 外部协作者有过期时间。
- 管理员变更告警。
- API token 需要 owner 和过期时间。

配置基线要自动化检查，否则随着团队扩张会逐渐漂移。

## 8. 供应商风险

供应商评估应包含：

- 安全认证和审计报告。
- 漏洞响应流程。
- 数据子处理方。
- 加密和密钥管理。
- 事件通知 SLA。
- 数据删除承诺。
- 业务连续性。

供应商风险不能完全外包给供应商。内部仍然要控制账号、权限和数据使用方式。

## 9. 退出策略

每个重要 SaaS 都应有退出方案：

- 数据导出格式。
- 导出权限和流程。
- 历史审计日志保留。
- 用户和集成下线步骤。
- 合同终止后的数据删除证明。
- 替代系统迁移计划。

接入前就要确认退出策略，采购结束再补会很被动。

## 10. 成熟度指标

| 指标 | 目标 |
| --- | --- |
| SSO 覆盖率 | 高风险 SaaS 100% |
| SCIM 覆盖率 | 核心 SaaS 优先 |
| 管理员数量 | 持续最小化 |
| 审计日志接入率 | 高风险 SaaS 100% |
| 高风险 OAuth 应用数量 | 可解释且受控 |
| 外部共享过期率 | 持续提升 |

SaaS 安全要把外部产品纳入内部安全控制面，不能只把风险写进合同。
