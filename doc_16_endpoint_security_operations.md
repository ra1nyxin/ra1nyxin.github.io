# Endpoint Security Operations

终端安全要让设备状态、执行行为、权限变化、数据访问和响应动作形成闭环，安装 EDR 只是起点。终端是身份、数据和攻击执行链的交汇点，很多事件最终都要回到终端证据上确认。

## 1. 终端资产

每台终端至少要知道：

| 字段 | 说明 |
| --- | --- |
| device_id | 稳定设备标识 |
| owner | 使用人或责任团队 |
| os_version | 操作系统版本 |
| security_agent | 安全代理状态 |
| encryption | 磁盘加密状态 |
| patch_level | 补丁状态 |
| privilege | 本地管理员情况 |

没有资产上下文的终端告警，很难判断严重性。

## 2. 基线控制

推荐基线：

- 磁盘加密默认开启。
- 禁止普通用户长期拥有本地管理员权限。
- 安全代理不可被普通管理员关闭。
- 脚本执行策略有审计。
- USB 和外设策略按数据等级配置。
- 浏览器、办公软件和远程工具保持更新。

基线不要只发文档，要能自动检查和报告漂移。

## 3. 高价值遥测

终端检测优先关注：

| 遥测 | 用途 |
| --- | --- |
| Process creation | 还原执行链 |
| Command line | 判断意图和参数 |
| Parent process | 识别异常启动关系 |
| Network connection | 发现外联和横向访问 |
| File modification | 发现持久化和数据打包 |
| Registry / Launch item | 发现自启动 |
| User logon | 关联身份和设备 |

采集命令行时要注意 secret 和隐私脱敏。

## 4. 本地管理员

本地管理员是终端横向移动的重要放大器。

控制建议：

- 默认不授予本地管理员。
- 临时提权有审批和过期时间。
- 本地管理员密码唯一化和轮换。
- 管理员提权事件进入日志。
- 高风险操作触发重新认证。

长期本地管理员权限越多，终端失陷后的影响越难收敛。

## 5. 响应动作

常见动作：

| 动作 | 使用场景 |
| --- | --- |
| Network isolation | 主机疑似失陷但需要保留证据 |
| Kill process | 明确恶意进程仍在运行 |
| Quarantine file | 文件可疑但需保留样本 |
| Collect triage package | 需要快速调查 |
| Reimage | 可信恢复比清理更可靠 |
| Credential reset | 账号或 token 可能泄露 |

不要在没有证据保全的情况下直接清理，尤其是高严重事件。

## 6. 检测场景

优先检测：

- Office 或浏览器子进程启动脚本解释器。
- 普通用户设备创建新本地管理员。
- 非管理工具发起远程连接。
- 终端短时间访问大量内部主机。
- 压缩大量敏感目录后外联。
- 安全代理停止或配置被修改。

终端检测要结合用户、设备、进程链和网络上下文，单点命中通常误报较高。

## 7. 度量指标

| 指标 | 目标 |
| --- | --- |
| EDR 覆盖率 | 托管终端接近 100% |
| 磁盘加密覆盖率 | 100% |
| 本地管理员比例 | 持续下降 |
| 高危补丁超期设备 | 接近 0 |
| 安全代理离线设备 | 可解释 |
| 终端隔离平均时间 | 持续降低 |

终端安全的成熟标志是：设备状态可见，异常行为可查，响应动作可执行。
