# Browser and Client Side Security

浏览器是用户访问企业系统、SaaS 和互联网的主要工作台。客户端安全不仅包括 XSS，还包括扩展、下载、剪贴板、同源策略、存储、第三方脚本和浏览器配置。

## 1. 浏览器管理

企业浏览器基线：

- 自动更新。
- 禁止不受信扩展。
- 管理密码保存策略。
- 启用安全浏览。
- 控制下载高风险文件。
- 隔离高风险站点。
- 配置证书和代理策略。

浏览器过期会让大量 Web 控制失效。

## 2. 扩展风险

扩展可以访问页面内容、cookie、请求和剪贴板。

控制建议：

- 扩展 allowlist。
- 禁止用户安装未知扩展。
- 审查扩展权限。
- 监控扩展 owner 和更新。
- 对高权限岗位使用更严格策略。

恶意或被收购的扩展可能成为数据泄露入口。

## 3. Web 存储

前端存储风险：

- localStorage 中保存 token。
- sessionStorage 保存敏感数据。
- IndexedDB 长期保存业务数据。
- 缓存中残留敏感响应。

高价值 token 应尽量使用安全 cookie 或短期内存态方案，并配合服务端会话控制。

## 4. 第三方脚本

第三方脚本能在页面上下文执行。

控制：

- 最小化第三方脚本。
- 使用 CSP。
- 对关键脚本启用 SRI。
- 审查供应商用途。
- 禁止在敏感页面加载非必要脚本。

支付、登录和管理页面应尤其谨慎。

## 5. CSP 设计

CSP 不是复制模板。

建议：

- 从 report-only 开始。
- 收集违规报告。
- 收紧 script-src。
- 避免 unsafe-inline。
- 限制 connect-src。
- 对敏感页面使用更严格策略。

CSP 能降低 XSS 影响，但不能替代输入和输出处理。

## 6. 下载和文件处理

浏览器下载风险：

- 用户下载恶意附件。
- 文件名混淆。
- MIME 类型错误。
- HTML 文件被本地打开执行脚本。
- 内部文件下载越权。

服务端应设置正确 Content-Type、Content-Disposition 和 X-Content-Type-Options。

## 7. 检测场景

优先关注：

- 新增高权限扩展。
- 用户访问已知钓鱼站点。
- 浏览器下载可执行文件后出现异常进程。
- 管理页面 CSP 违规激增。
- 第三方脚本域名变化。

浏览器信号和终端、身份日志结合后价值更高。

## 8. 指标

| 指标 | 目标 |
| --- | --- |
| 浏览器自动更新覆盖 | 100% |
| 未批准扩展数量 | 0 |
| 高风险页面 CSP 覆盖 | 100% |
| 第三方脚本清单 | 完整 |
| 钓鱼访问阻断率 | 持续提升 |

浏览器安全的核心是减少客户端执行和数据暴露的不确定性。
