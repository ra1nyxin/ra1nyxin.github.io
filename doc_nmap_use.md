# Nmap 常用用法

这篇放一些我平时会反复查的 Nmap 命令。重点是快速确认目标状态、端口、服务版本和输出结果，适合在授权环境里的资产梳理、服务排查和基础安全验证。

## 快速确认主机是否在线

只做主机发现，速度快，适合先扫一个网段里有哪些机器活着。

```bash
nmap -sn 192.168.1.0/24
```

遇到 ICMP 被拦的环境，可以跳过主机发现，直接把目标当成在线处理。

```bash
nmap -Pn 192.168.1.10
```

## 常用端口扫描

默认扫描常见 TCP 端口，适合先看一个目标的大概暴露面。

```bash
nmap 192.168.1.10
```

指定端口范围，排查某一段端口时更省时间。

```bash
nmap -p 1-10000 192.168.1.10
```

扫描全部 TCP 端口，耗时会更长，结果也更完整。

```bash
nmap -p- 192.168.1.10
```

扫几个明确端口，用来验证服务是否还开着。

```bash
nmap -p 22,80,443,8080 192.168.1.10
```

## 服务和版本识别

确认端口后再做服务识别，结果会比单纯看端口号可靠。

```bash
nmap -sV -p 22,80,443 192.168.1.10
```

加上默认脚本和系统识别，适合做一次稍完整的服务画像。

```bash
nmap -sV -sC -O 192.168.1.10
```

版本识别强度调高一点，适合服务 banner 很模糊的时候。

```bash
nmap -sV --version-intensity 8 192.168.1.10
```

## UDP 扫描

UDP 扫描慢，建议先挑常见端口。

```bash
nmap -sU -p 53,67,68,69,123,161,500,4500 192.168.1.10
```

只看前 100 个常见 UDP 端口，适合粗略确认。

```bash
nmap -sU --top-ports 100 192.168.1.10
```

## NSE 脚本

查看某个服务相关脚本时，可以先搜脚本目录。

```bash
nmap --script-help http-title
```

跑 HTTP 标题识别，常用于快速确认 Web 服务内容。

```bash
nmap --script http-title -p 80,443,8080 192.168.1.10
```

跑默认安全检查脚本，适合做基础探测。

```bash
nmap --script default,safe 192.168.1.10
```

## 输出保存

保存三种常用格式，后面整理报告或继续处理都方便。

```bash
nmap -sV -oA scan-result 192.168.1.10
```

只保存普通文本。

```bash
nmap -sV -oN scan-result.txt 192.168.1.10
```

保存 XML，后续交给别的工具处理会更顺手。

```bash
nmap -sV -oX scan-result.xml 192.168.1.10
```

## 常见节奏

我一般会先做主机发现，再扫全端口，然后针对开放端口做服务识别。遇到防火墙或延迟很高的网络，再调整速率和重试次数。

```bash
nmap -p- --min-rate 3000 -oN ports.txt 192.168.1.10
```

```bash
nmap -sV -sC -p 22,80,443 -oN services.txt 192.168.1.10
```

降低重试次数，适合内网里比较稳定的目标。

```bash
nmap --max-retries 2 -p- 192.168.1.10
```

## 备注

端口扫描结果只说明当时看到的网络状态。目标有安全组、防火墙、负载均衡或端口敲门机制时，结果会跟实际服务部署有差异。重要资产建议从不同网络位置各扫一次，再和资产清单、云平台安全组一起对照。
