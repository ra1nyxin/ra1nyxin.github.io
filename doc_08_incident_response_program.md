# Incident Response Program

安全事件响应需要可演练、可复盘、可持续改进的组织能力。真正的响应能力取决于准备阶段，临时拉群和现场救火只能解决一部分问题。

## 1. 响应目标

事件响应有三个核心目标：

1. 降低业务影响。
2. 阻止攻击继续扩散。
3. 保留足够证据用于根因分析和改进。

不同事件的优先级不同。勒索、数据泄露、身份接管和供应链投毒不应该使用同一套节奏。

## 2. 事件分级

| 等级 | 示例 | 响应要求 |
| --- | --- | --- |
| SEV-1 | 生产环境大范围失陷、敏感数据泄露 | 立即升级，管理层和法务介入 |
| SEV-2 | 单个关键系统被入侵、高权限账号异常 | 安全、运维、业务 owner 联合处置 |
| SEV-3 | 可疑行为、局部恶意软件、普通账号泄露 | 安全团队主导调查 |
| SEV-4 | 低风险误报、扫描噪声 | 记录和优化规则 |

分级标准必须提前定义，否则事件中会消耗大量时间争论严重性。

## 3. 响应阶段

| 阶段 | 目标 |
| --- | --- |
| Preparation | 准备联系人、权限、工具、日志和预案 |
| Identification | 确认事件是否真实、范围多大 |
| Containment | 控制扩散，保护关键业务 |
| Eradication | 清除攻击者访问路径和持久化点 |
| Recovery | 恢复服务并持续监控 |
| Lessons learned | 复盘根因和改进措施 |

不要跳过准备阶段。没有日志、权限和联系人，响应现场会变成猜测。

## 4. 角色分工

| 角色 | 职责 |
| --- | --- |
| Incident commander | 协调节奏、决策优先级 |
| Security analyst | 调查证据、判断攻击路径 |
| Platform owner | 执行隔离、回滚、修复 |
| Business owner | 判断业务影响 |
| Legal / Compliance | 评估通知义务和合规风险 |
| Communications | 对内对外沟通 |

事件中需要一个明确的 commander。多人同时指挥会造成重复操作和遗漏。

## 5. 证据保全

事件响应中要尽量避免破坏证据。

优先保留：

- 原始告警和时间线。
- 登录日志、云审计日志、终端遥测。
- 可疑文件 hash 和路径。
- 进程、网络连接、计划任务、启动项。
- 相关账号权限和最近变更。
- 受影响资源快照。

证据记录应包含采集人、时间、来源、处理方式和完整性校验。

## 6. 常见处置动作

| 风险 | 动作 |
| --- | --- |
| 账号泄露 | 撤销会话、重置凭证、复核 MFA 和 OAuth 授权 |
| 主机失陷 | 网络隔离、采集镜像、清理持久化、重装或回滚 |
| 云密钥泄露 | 禁用密钥、轮换凭证、审计调用历史 |
| 数据外传 | 确认数据范围、保全访问日志、评估通知义务 |
| CI/CD 被篡改 | 冻结构建、轮换发布凭证、验证产物完整性 |

不要只修补入口漏洞。攻击者可能已经建立了新的访问路径。

## 7. 时间线方法

事件分析建议维护统一时间线：

```text
T0 首个异常信号
T1 初始访问
T2 权限扩张
T3 横向移动
T4 数据访问或破坏行为
T5 检测和响应动作
T6 完成遏制
```

时间线能帮助团队区分“已经发生的影响”和“仍然存在的风险”。

## 8. 复盘输出

复盘不应该停留在“加强安全意识”。

一份有效复盘至少包含：

- 根因。
- 影响范围。
- 哪些检测有效。
- 哪些日志缺失。
- 哪些流程阻塞响应。
- 永久修复项。
- 负责人和截止日期。
- 防止复发的验证方式。

复盘要聚焦系统改进，避免变成个人追责。

## 9. 演练计划

建议每季度做桌面演练，每半年做技术演练。

演练主题可以包括：

- 管理员账号被钓鱼。
- 云访问密钥泄露。
- 生产容器被植入后门。
- SaaS 数据被批量导出。
- CI/CD 流水线被篡改。
- 勒索软件影响核心文件服务。

演练结束后要更新 runbook、联系人、权限和检测规则。

## 10. 成熟度指标

| 指标 | 含义 |
| --- | --- |
| MTTD | 平均发现时间 |
| MTTA | 平均确认时间 |
| MTTC | 平均遏制时间 |
| MTTR | 平均恢复时间 |
| Runbook 覆盖率 | 常见事件是否有标准流程 |
| 演练问题关闭率 | 演练发现是否真正修复 |

响应能力的本质是把混乱变成可控，把未知变成证据，把一次事故变成体系改进。
