# Phishing Resilience Program

钓鱼防护需要持续运行，年度培训只能覆盖很小一部分。有效的钓鱼韧性需要技术控制、流程控制、用户报告、身份安全和持续演练共同工作。

## 1. 目标

钓鱼项目应关注：

- 减少钓鱼邮件到达用户。
- 降低用户提交凭证概率。
- 让用户能快速报告。
- 点击后快速确认账号风险。
- 推动流程和技术改进。

不要只用点击率评价用户。更重要的是报告率、响应时间和账号保护效果。

## 2. 技术控制

| 控制 | 作用 |
| --- | --- |
| FIDO2 / Passkey | 降低凭证钓鱼成功率 |
| Conditional access | 异常登录要求强验证 |
| Mail gateway | 阻断恶意链接和附件 |
| Browser isolation | 降低高风险链接影响 |
| URL filtering | 阻止访问恶意站点 |
| Session revocation | 点击后快速止损 |

抗钓鱼 MFA 是高权限账号的优先控制。

## 3. 演练设计

演练要真实但可控：

- 不使用羞辱式公开排名。
- 不模拟极端敏感个人场景。
- 按岗位设计场景。
- 记录报告行为。
- 结束后给出简短反馈。
- 将结果用于改进控制。

钓鱼演练的目标是训练组织，避免演练变成惩罚个人。

## 4. 评估指标

| 指标 | 含义 |
| --- | --- |
| Delivery rate | 到达率 |
| Click rate | 点击率 |
| Credential submit rate | 凭证提交率 |
| Report rate | 报告率 |
| Time to report | 报告耗时 |
| Time to containment | 遏制耗时 |

报告率上升通常是好信号，说明用户愿意参与防御。

## 5. 高风险人群

需要重点保护：

- 管理员。
- 财务和采购。
- HR。
- 高管助理。
- 开发和运维。
- 对外沟通频繁的岗位。

高风险人群应优先启用强 MFA、额外提示和定向训练。

## 6. 点击后流程

用户点击可疑链接后：

1. 收集邮件和 URL。
2. 查询是否提交凭证。
3. 检查登录日志和 MFA 事件。
4. 撤销可疑会话。
5. 检查邮箱规则和转发。
6. 评估是否影响其他用户。

不要让点击事件停留在“提醒用户小心”。

## 7. 流程防护

钓鱼常绕过技术，攻击业务流程。

关键流程要设置：

- 收款账号变更双人确认。
- 供应商联系人变更回拨。
- 高额付款审批。
- 敏感数据导出审批。
- 账号恢复二次验证。

流程控制是 BEC 防护的核心。

## 8. 成熟度

| 阶段 | 特征 |
| --- | --- |
| Basic | 有邮件网关和年度培训 |
| Managed | 有报告按钮和演练 |
| Integrated | 邮件、身份、响应联动 |
| Resilient | 抗钓鱼 MFA 和流程防护覆盖关键岗位 |

成熟钓鱼防护看组织韧性，单次演练成绩只能做参考。
