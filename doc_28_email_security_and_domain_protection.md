# Email Security and Domain Protection

邮件仍然是身份攻击、供应链欺诈和恶意链接投递的主要入口。邮件安全不只依赖网关，还需要域名保护、身份联动、用户报告、检测响应和供应商治理。

## 1. 域名认证

核心机制：

| 机制 | 作用 |
| --- | --- |
| SPF | 声明允许发信的服务器 |
| DKIM | 对邮件内容和头部签名 |
| DMARC | 指定认证失败处理策略 |
| BIMI | 品牌标识，依赖强 DMARC |

DMARC 策略应从监控逐步推进到 quarantine 或 reject。

## 2. 发信源治理

要维护所有合法发信源：

- 企业邮箱。
- 营销平台。
- 工单系统。
- 账单系统。
- CI/CD 通知。
- 第三方业务系统。

未知发信源会让 SPF 和 DMARC 策略难以收紧。

## 3. 防钓鱼控制

重点控制：

- URL 重写和沙箱分析。
- 附件沙箱。
- 高风险发件人标记。
- 外部邮件提示。
- 相似域名监控。
- 登录链接风险分析。
- 用户报告按钮。

邮件网关应和身份平台联动，钓鱼点击后能触发会话撤销和账号检查。

## 4. BEC 风险

商业邮件欺诈通常利用流程和信任，而不是恶意附件。

高风险场景：

- 修改收款账号。
- 紧急付款请求。
- 高管冒充。
- 供应商联系人变更。
- 采购和发票异常。

控制要结合财务流程，例如双人确认、可信回拨和变更冷却期。

## 5. 用户报告流程

用户报告可疑邮件后，应自动：

- 提取邮件头和附件 hash。
- 判断是否已影响其他用户。
- 查询是否有人点击链接。
- 创建调查记录。
- 根据结果批量清理邮箱。

让用户报告后没有反馈，会降低后续报告意愿。

## 6. 检测场景

优先检测：

- DMARC 失败的大量邮件。
- 相似域名发来的登录通知。
- 新供应商域名发送付款请求。
- 用户点击高风险链接后登录异常。
- 邮箱规则被创建用于隐藏邮件。
- 邮件转发到外部地址。

邮件安全要把邮件事件、身份日志和 SaaS 日志串起来。

## 7. 响应动作

常见动作：

- 撤回或隔离邮件。
- 阻断 URL 或附件 hash。
- 重置账号凭证。
- 撤销 refresh token。
- 检查邮箱规则和转发。
- 通知受影响用户。
- 更新网关策略。

钓鱼响应不能只删除邮件，必须确认账号是否被接管。

## 8. 指标

| 指标 | 目标 |
| --- | --- |
| DMARC reject 覆盖 | 主域名优先 |
| 合法发信源清单 | 完整 |
| 用户报告处理时间 | 持续降低 |
| 点击后账号检查覆盖 | 100% |
| 外部转发异常 | 可调查 |
| 相似域名处置时间 | 持续降低 |

邮件安全的重点是降低账号接管概率，并让钓鱼事件能快速收敛。
