# External Attack Surface Management

外部攻击面管理关注组织在互联网上真实暴露的资产，包括域名、IP、云服务、存储、证书、第三方托管页面、API、测试环境和遗留系统。

攻击者看到的是外部可发现入口。安全团队需要用同样视角持续观察，内部资产台账只能作为辅助。

## 1. 发现对象

| 对象 | 风险 |
| --- | --- |
| 域名和子域名 | 遗留系统、接管风险 |
| IP 和端口 | 未授权服务暴露 |
| TLS 证书 | 资产线索和过期风险 |
| Cloud storage | 公开数据泄露 |
| API endpoint | 未纳管接口 |
| Login portal | 爆破、钓鱼目标 |
| Test environment | 弱配置和真实数据 |

外部暴露面会频繁变化，不能只靠季度扫描。

## 2. 资产归属

每个外部资产必须能回答：

- 属于哪个业务。
- 谁是技术 owner。
- 是否仍然需要暴露。
- 承载什么数据。
- 是否接入日志和监控。
- 是否有漏洞管理。

无法确认归属的外部资产应进入高优先级排查。

## 3. 暴露面基线

建议定义：

- 允许公开的端口。
- 允许公开的管理界面。
- TLS 最低版本。
- 禁止公开的存储类型。
- 测试环境暴露规则。
- 登录入口安全要求。

基线要配合自动发现，否则会很快失效。

## 4. 重点风险

| 风险 | 说明 |
| --- | --- |
| Subdomain takeover | DNS 指向失效第三方资源 |
| Exposed admin panel | 管理界面暴露公网 |
| Forgotten staging | 测试环境无人维护 |
| Public bucket | 对象存储权限错误 |
| Expired certificate | 可用性和信任风险 |
| Shadow API | 未纳入安全测试的 API |

这些问题多数来自资产治理缺口，高级攻击反而少见。

## 5. 扫描频率

建议：

- 高风险域名每日发现。
- 关键端口每日差异扫描。
- 证书和 DNS 每日监控。
- 全量外部资产每周复核。
- 重大云账号变更后立即扫描。

扫描结果要进入工单和 owner 流程，不要停留在报表。

## 6. 变更检测

需要关注：

- 新增子域名。
- 新开放端口。
- 新证书签发。
- DNS 指向新供应商。
- 存储权限变更为公开。
- 登录入口新增。

外部攻击面管理重点看新变化，重复列出已知资产价值有限。

## 7. 与漏洞治理联动

外部暴露资产的漏洞优先级应提高，特别是：

- 已知可利用漏洞。
- 未认证远程访问。
- 管理接口。
- 承载敏感数据的服务。
- 没有 WAF 或访问控制的 API。

同样 CVSS 的漏洞，在公网和内网的风险不同。

## 8. 指标

| 指标 | 目标 |
| --- | --- |
| 未归属外部资产 | 0 |
| 未批准公开端口 | 0 |
| 公开存储桶 | 0 |
| 过期证书 | 0 |
| 新暴露资产确认时间 | 持续降低 |
| 外部高危漏洞超期 | 0 |

外部攻击面管理的目标是让互联网暴露变成受控变更，减少偶然发现。
