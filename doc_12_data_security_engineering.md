# Data Security Engineering

数据安全不是在数据库外面加一圈权限就结束了。真实系统中的数据会流经采集、传输、存储、处理、导出、备份、分析、AI 检索和第三方集成。每一个环节都可能改变风险。

## 1. 数据分类

先定义可执行的数据分类。

| 级别 | 示例 | 控制要求 |
| --- | --- | --- |
| Public | 公开文档、公开页面 | 完整性保护 |
| Internal | 内部流程、普通业务数据 | 访问控制和审计 |
| Confidential | 客户数据、财务数据、商业策略 | 加密、最小权限、导出控制 |
| Restricted | 密钥、身份凭证、敏感个人信息 | 强隔离、严格审批、细粒度审计 |

分类必须和系统字段、数据表、对象存储路径或数据集绑定，否则只是标签。

## 2. 数据生命周期

| 阶段 | 风险 | 控制 |
| --- | --- | --- |
| Collection | 过度收集 | 数据最小化、告知和授权 |
| Storage | 未授权访问 | 加密、ACL、密钥管理 |
| Processing | 越权计算 | 任务身份、隔离执行 |
| Sharing | 第三方滥用 | 合同、审计、脱敏 |
| Export | 批量外传 | 审批、限速、水印 |
| Retention | 长期保留 | 到期删除、归档策略 |
| Deletion | 删除不彻底 | 删除证明、备份策略 |

数据安全要覆盖生命周期，而不是只覆盖数据库。

## 3. 访问模型

推荐按数据和动作建模：

```text
subject: 用户或服务
dataset: 数据集
record scope: 租户、区域、业务线
action: read、write、export、delete、admin
purpose: 使用目的
duration: 授权时长
```

访问敏感数据时，读取和导出应该是不同权限。很多泄露事件不是因为能看，而是因为能批量导出。

## 4. 加密与密钥

加密要关注密钥控制面。

| 项目 | 要求 |
| --- | --- |
| Data at rest | 存储层加密 |
| Data in transit | TLS，禁止明文传输 |
| Key ownership | 明确 key owner |
| Key rotation | 定期轮换 |
| Key access logs | 记录使用和管理操作 |
| Separation | 密钥权限与数据权限分离 |

如果拿到应用管理员权限就能直接读取密钥和数据，加密只能提供有限保护。

## 5. 脱敏与匿名化

常见技术：

| 方法 | 适用场景 |
| --- | --- |
| Masking | 展示部分字段，例如手机号后四位 |
| Tokenization | 用代号替换敏感值 |
| Hashing | 去重或匹配，不需要还原 |
| Aggregation | 只保留统计结果 |
| Differential privacy | 大规模分析中降低重识别风险 |

脱敏要基于使用场景设计。测试环境通常不需要真实敏感数据。

## 6. 数据导出控制

导出是高风险动作。

建议控制：

- 大批量导出需要审批。
- 导出任务记录操作者、范围、理由。
- 导出文件设置有效期。
- 敏感导出加水印。
- 异常导出速率触发告警。
- 第三方共享有合同和审计要求。

导出能力应该被视为独立权限，而不是读取权限的自然延伸。

## 7. 数据发现

数据安全需要先知道数据在哪里。

发现机制包括：

- 扫描数据库字段和对象存储。
- 识别 PII、secret、凭证和业务敏感字段。
- 标记数据 owner。
- 记录数据流向。
- 发现未登记数据集。

数据发现结果要接入权限复核和删除流程，否则会变成一次性盘点。

## 8. AI 与数据安全

AI 系统会引入新的数据路径：

- Prompt 中包含敏感信息。
- RAG 索引保存内部文档。
- 模型日志记录用户输入。
- 工具调用访问企业数据。
- 训练集混入未授权数据。

AI 数据控制要求：

- Prompt 和日志脱敏。
- 检索索引按租户和权限隔离。
- 高风险工具调用做策略检查。
- 明确保留期和删除机制。
- 禁止低信任内容驱动高权限数据操作。

## 9. 检测场景

可以优先建立这些检测：

- 普通用户短时间读取大量敏感记录。
- 非业务时间批量导出。
- 新创建 API token 立即访问敏感数据。
- 第三方集成访问范围突然扩大。
- 数据库快照被复制到低安全区域。
- 对象存储 bucket 权限变为公开。

数据安全检测必须和数据分类联动，否则很难判断严重性。

## 10. 度量指标

| 指标 | 目标 |
| --- | --- |
| 数据集 owner 覆盖率 | 100% |
| 敏感字段标记率 | 持续提升 |
| 未授权公开数据集 | 0 |
| 导出审批覆盖率 | 高风险数据 100% |
| 测试环境真实敏感数据 | 接近 0 |
| 数据删除 SLA 达成率 | 可证明 |

数据安全的最终目标是让数据被正确使用，而不是让数据无法使用。
